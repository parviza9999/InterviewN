## 3. Dataset Options Reviewed

The team reviewed two possible datasets for the fraud detection capstone project.

### Option 1: Credit Card Fraud Detection Dataset

Option 1 contains anonymized European credit card transactions. It is highly imbalanced and is well suited for fraud detection, anomaly detection, baseline model comparison, supervised classification, and end-to-end deployment.

### Option 2: Bank Account Fraud / BAF Suite

Option 2 is a more advanced dataset focused on bank account fraud and fairness/bias analysis. Although it is a strong dataset, it adds additional complexity that is outside the scope of this capstone project.

## 4. Dataset Selection Decision

The team selected **Option 1: Credit Card Fraud Detection Dataset** as the only dataset for this capstone project.

Option 1 was selected because it supports the full project lifecycle: data loading, exploratory data analysis, class imbalance analysis, automated preprocessing, baseline modeling, Autoencoder-based anomaly detection, XGBoost supervised classification, model evaluation, and cloud deployment.

Option 2, the Bank Account Fraud / BAF Suite, was reviewed but excluded from the project scope. The team decided that using one dataset only would make the project more focused, achievable, and suitable for an end-to-end deployment within the available timeline.

## 10. Next Phase Plan

The next phase will focus entirely on Option 1, the Credit Card Fraud Detection dataset.

The team will load the dataset, complete exploratory data analysis, document class imbalance, check for missing values, summarize feature distributions, and prepare an automated preprocessing pipeline.

After the EDA and preprocessing work is complete, the project will move into baseline model development, Autoencoder anomaly detection, XGBoost model comparison, model evaluation, and cloud deployment.
